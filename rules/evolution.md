# 自我进化机制

> 本技能具备自我进化与最优决策能力，能从每次交互中学习并持续优化处理策略。
> 
> **默认决策**：无法明确判定时，默认采用**小说模式 + 完整降味处理**。

### 学习规则

| 场景 | 学习行为 |
|------|----------|
| 同一文本被用户反复发送 | 说明上次处理不理想，调整策略 |
| 用户主动要求重做 | 记录失败模式，下次避免 |
| 用户使用更精细指令（如"不要删我的'感到'"） | 更新个性化规则 |
| 用户明确表扬 | 强化该处理模式 |
| 用户手动修改后满意 | 学习用户的修改偏好 |
| 用户纠正文体判定 | 更新文体识别规则 |

## 二、最优决策算法

### 文体自动识别决策树

```
收到文本
    │
    ├─ 用户指定了快捷指令 → 直接应用对应规则
    │   ├─ /novel、/wechat → 执行排版+标点+完整降味
    │   ├─ /report、/paper → 执行排版+标点（不降味）
    │   ├─ /punc → 只执行P0标点修复
    │   ├─ /para → 只执行P1段落调整
    │   ├─ /humanize → 只执行AI降味规则
    │   └─ /auto → 进入自动分析模式
    │
    ├─ 用户使用"/auto"或未指定指令 → 进入自动分析模式
    │   ├─ 检测到对话格式+感知动词（"感到""然后"） → 判定为小说 → 应用小说规则+完整降味
    │   ├─ 检测到"首先其次"+短段落+高频AI词 → 判定为公众号 → 应用公众号规则+完整降味
    │   ├─ 检测到"一、二、三"标题+长段落+无AI词 → 判定为报告 → 应用报告规则（不降味）
    │   ├─ 检测到参考文献/摘要/关键词+学术语气 → 判定为论文 → 应用论文规则（不降味）
    │   └─ 无法判定或无明确特征 → **默认小说规则+完整降味**
    │
    └─ 未指定任何指令 → 默认执行/auto（自动分析，默认小说+降味）
```

### 处理深度决策

```
文本长度
    │
    ├─ < 200字 → P0 + 基础降味（删除高频词和感知动词）
    │
    ├─ 200-1000字 → P0+P1 + 基础降味
    │
    └─ > 1000字 → P0+P1+P2 + 完整降味（所有降味规则）
```

### 降味执行决策

| 文体 | 快捷指令 | 是否降味 | 降味级别 |
|------|----------|----------|----------|
| 小说 | /novel | ✅ 是 | 完整降味 |
| 公众号 | /wechat | ✅ 是 | 完整降味 |
| 报告 | /report | ❌ 否 | 不降味 |
| 论文 | /paper | ❌ 否 | 不降味 |
| 仅标点 | /punc | ❌ 否 | 不降味 |
| 仅段落 | /para | ❌ 否 | 不降味 |
| 仅降味 | /humanize | ✅ 是 | 完整降味 |
| 自动 | /auto | 视判定 | 小说/公众号=完整，报告/论文=不降 |

### 冲突决策优先级

当规则冲突时，按以下优先级裁决：

1. **用户明确指令** > 用户习惯 > 文体默认规则 > 通用规则
2. **小说/公众号场景**：降味优先于格式规范
3. **报告/论文场景**：规范优先于降味（实际上不执行降味）
4. **不确定时**：保守处理（不改），并采用**默认小说+降味**

## 三、策略进化机制

### 规则有效性评分

每处理100次自动评估：

| 指标 | 计算公式 | 目标值 |
|------|----------|--------|
| 标点修复准确率 | 正确修复数 / 应修复总数 | ≥95% |
| 段落拆分合理率 | 用户未要求重拆的比例 | ≥90% |
| 文体识别准确率 | 用户手动纠正的比例 | ≥85% |
| **降味接受率** | 用户未要求恢复原表达的比例 | ≥85% |
| **降味自然度** | 用户反馈"像人写的"比例 | ≥80% |

### 自动优化规则

| 条件 | 行动 |
|------|------|
| 准确率 < 90% | 标记为"待优化"，触发规则审查 |
| 用户手动纠正 | 自动记录为"例外规则"，下次优先匹配 |
| 同一规则连续3次准确率 > 95% | 提升为"核心规则"，执行优先级提高 |
| 同一错误模式出现5次 | 创建针对性修复规则 |
| **降味被频繁撤销** | 标记该降味规则为"激进"，调整为保守模式 |

### 规则库版本管理

- 每次成功处理更新本地规则权重
- 累计100次处理后输出进化报告
- 用户可通过 `/learn` 查看当前规则版本和统计数据
- **降味规则单独追踪效果**

## 六、重置机制

用户输入 `/reset` 时：

1. 清除所有学习记录
2. 恢复出厂规则
3. 重置统计数据
4. **恢复默认设置：小说+降味**
5. 输出确认信息

```
【重置确认】
已恢复出厂设置：
- 学习记录已清除
- 规则权重已重置
- 统计数据已清零
- 个性化偏好已删除
- 默认设置：小说+降味

当前版本：1.5（出厂状态）
```

## 八、默认决策说明

### 为什么默认小说+降味？

1. **小说对AI痕迹最敏感**：读者对"人味儿"要求最高
2. **降味收益最大**：小说降味前后差异最明显
3. **安全保守**：降味可随时撤销，不执行降味可能导致用户不满意
4. **符合用户预期**：大多数用户使用本技能是为了让文本更像人写的

### 何时不执行默认？

- 用户明确指定 `/report` 或 `/paper`
- 自动检测到明确的报告/论文特征
- 用户历史记录显示偏好不降味
